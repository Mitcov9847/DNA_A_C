# Лабораторная работа №9  
## Тема: Nullability

---

## Вопросы и ответы

### **1. Что такое `null`?**
`null` — это специальное значение, означающее отсутствие ссылки (адреса памяти).  
Переменная-ссылка не указывает ни на какой объект.

---

### **2. Почему тип `int` не позволяет значение `null`?**
`int` — это тип-значение.  
Число `0` — это просто число, а не отсутствие значения.  
Поэтому `int` сам по себе не может хранить `null`.

---

### **3. Что означает `string?`?**
- `string` — ссылочный тип.  
- `string?` — строка, которой разрешено иметь значение `null`.

Компилятор начинает проверять, правильно ли вы учитываете возможный `null`.

---

### **4. Почему `int?` и `string?` работают по-разному? Как работает `int?`?**
- `string?` просто указывает, что строка может быть `null` (это естественно для ссылок).
- `int?` превращается в структуру `Nullable<int>`, которая содержит:
  - `Value` — значение типа `int`
  - `HasValue` — флаг, есть ли значение

То есть тип-значение получает возможность хранить `null`.

---

### **5. Что делает оператор `!` (null-forgiving operator)?**
Оператор `!` говорит компилятору «здесь точно не null, поверь мне».

```
static string? MaybeString(int num)
{
    return num == 0 ? "Hello" : null;
}
string s = MaybeString(0)!;
```

### 6. Пример обхода nullability с помощью !
```
// Функция вернёт null, но ошибки компиляции не будет
string s = MaybeString(1)!;
```

```
string s = null!;
```
Это показывает, что оператор ! отключает защиту компилятора.

### 7. Для чего нужен required?
required заставляет обязательно инициализировать свойство при создании объекта.

```
class User
{
    public required string Name { get; set; }
}
```
Если не указать Name — проект не скомпилируется.

Задание
###### 1. Включите nullability в проекте
В файл .csproj добавить:

```
<Nullable>enable</Nullable>
```
###### 2. Сделайте так, чтобы предупреждения были ошибками
Добавить:

<TreatWarningsAsErrors>true</TreatWarningsAsErrors>
Пример корректного .csproj
```
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net9.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>

    <TreatWarningsAsErrors>true</TreatWarningsAsErrors>
    <Nullable>enable</Nullable>
  </PropertyGroup>
</Project>
```
###### 3. Исправьте появившиеся предупреждения
 
После включения nullability необходимо:
Добавить ? к ссылочным типам, которые могут быть null
Инициализировать все обязательные значения
Минимизировать использование !
Проверять данные на null перед использованием
