# Инструкция по работе с Git и GitHub

Привет! Здесь я кратко рассказываю, как я загружаю свой проект на GitHub.

---

## 1. Создание локального репозитория

Сначала заходим в папку с проектом и выполняем:

```bash
git init
```

Это превращает мою папку в Git-репозиторий, чтобы я мог отслеживать изменения.

---

## 2. Добавляем `.gitignore`

Создаю файл `.gitignore`, чтобы Git не отслеживал лишние файлы, например:

```
*.log
*.tmp
*.exe
*.class
node_modules/
vendor/
```

Так я не буду загружать временные или сгенерированные файлы.

---

## 3. Настройка своих данных в Git

Настраиваю имя и почту для коммитов:

```bash
git config --global user.name "Евгений Митков"
git config --global user.email "JENIAMITCOV@email.com"
```

Теперь Git знает, кто делает коммиты.

---

## 4. Добавляем файлы и делаем первый коммит

```bash
git add .
git commit -m "Первый коммит: добавлены все файлы проекта и .gitignore"
```

`git add .` — добавляет все файлы, а `git commit` — сохраняет изменения.

---

## 5. Создаем репозиторий на GitHub

Заходим на GitHub, создаем новый репозиторий, даем ему имя, например `LAB03`, и нажимаем **Create repository**.

---

## 6. Связываем локальный репозиторий с GitHub

Берем URL репозитория с GitHub и выполняем:

```bash
git remote add origin https://github.com/
git branch -M main
git push --set-upstream origin main
```

Теперь мой проект загружен на GitHub и все изменения можно отслеживать и синхронизировать.
