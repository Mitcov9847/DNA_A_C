# Лабораторная работа 8: Улучшение программы библиотеки

## Описание

В этой лабораторной я создал простую систему управления библиотекой на языке C#. В библиотеке хранится несколько книг, каждая из которых имеет название, год выпуска и статус (взята она или доступна).

## Что было сделано

1. **Структура `Book`** — я использовал структуру, чтобы объединить данные о каждой книге в одном месте. Это удобно, потому что все свойства книги хранятся вместе, и с ними легко работать.

2. **Массив книг** — я создал массив, который содержит все книги библиотеки. Благодаря этому можно легко перебирать книги через цикл и выполнять над ними действия.

3. **Функции для операций с книгами**:

   * `ViewAll` — показывает все книги и их статус.
   * `Borrow` — позволяет взять книгу по названию, проверяя, взята она уже или нет.
   * `Return` — возвращает книгу в библиотеку, проверяя, была ли она взята.

   Вынесение этих операций в отдельные функции помогло убрать повторяющийся код и сделать программу более понятной.

4. **Меню с циклом `while`** — программа постоянно ждёт действия пользователя, пока он не решит выйти. Пользователь может выбрать просмотр всех книг, взятие книги или её возврат.

## Зачем это сделано

* Я использовал **структуру и массив**, чтобы хранить и обрабатывать несколько книг удобно и наглядно.
* Вынесение действий в **отдельные функции** делает код проще, легче исправлять ошибки и добавлять новые возможности.
* Использование **цикла в меню** позволяет программе работать постоянно и реагировать на действия пользователя без повторного запуска.

## Защита лабораторной работы №8

### 1. Краткое описание проекта
В работе реализована простая библиотечная система на C#. Программа позволяет просматривать список книг, брать книгу и возвращать её. Каждая книга содержит название, год выпуска и статус (взята или доступна).

### 2. Использование структуры `Book`
Для группировки данных о книге я использовал структуру `Book`. Это позволяет объединить связанные поля (название, год, статус) в один объект и упрощает работу с массивом книг.

### 3. Почему выбран массив
По условиям задания нужно хранить 5 книг. Массив идеален для фиксированного набора элементов и его удобно перебирать циклом.

### 4. Использование отдельных функций
Я вынес каждую операцию в отдельную функцию:

- `ViewAll` — вывод всех книг
- `Borrow` — взять книгу
- `Return` — вернуть книгу
- `Menu` — меню управления

Такой подход убирает повторяющийся код и делает программу более структурированной.

### 5. Использование циклов
Для поиска книги и вывода списка используются циклы `for` и `foreach`. Это позволяет проходить по массиву книг и выполнять нужные действия.

### 6. Обработка ошибок
Программой предусмотрена корректная обработка ошибок:
- если книги с указанным названием нет — выводится сообщение об ошибке;
- если книга уже взята — повторно взять её нельзя;
- если книга находится в библиотеке — вернуть её повторно невозможно.

### 7. Улучшение программы
Главное улучшение — разделение логики на функции.  
Это устраняет дублирование кода и делает программу легче для расширения.

### 8. Итог
Программа полностью соответствует требованиям лабораторной работы: использованы структуры, массив, циклы, вынесенные функции и реализованы все необходимые операции.
