# Лабораторная работа №6

**Тема: Алгоритмы**

---

## Что такое алгоритм

Алгоритм — это набор шагов, которые нужно сделать, чтобы решить задачу.
Он всегда имеет:

* что получаем на входе (ввод)
* что получаем в итоге (вывод)
* как будем использовать (интерфейс)
* как это делаем (реализация/код)

---

## Абстракция

Абстракция — это когда мы смотрим на задачу не в деталях, а в общих чертах.
Пример: «сделать обед» — это абстрактно, «почистить картошку» — конкретно.

---

## Декомпозиция

Сложные задачи проще решать, если делить на маленькие.
Пример: «сделать обед»:

1. Сделать первое блюдо
2. Сделать второе блюдо
3. Сделать напиток

Каждое из этого можно ещё разбить на шаги: взять продукты, подготовить посуду, готовить по рецепту.

---

## Списки и массивы

* Список или массив — набор элементов в памяти, один за другим
* У каждого есть номер (индекс), чтобы к нему обращаться
* Индексы обычно с 0

Пример:

```
5, 12, 7, 3, 9
0  1  2  3  4
```

---

## Как проходить по списку

Чтобы пройти все элементы, нужна переменная, которая показывает текущий элемент.
Чтобы посчитать сумму: нужна ещё переменная для суммы.

Пример:

```
sum = 0
для каждого i от 0 до длины списка - 1:
    sum = sum + список[i]
```

---

## Алгоритм, который вызывает другой

Пример: сортировка вызывает сравнение элементов
Или поиск пути вызывает проверку соседних вершин

---

## Практическая задача

**Дано:** два списка одинаковой длины
**Нужно:** посчитать попарные суммы элементов

### Шаги:

1. Получаем списки A и B
2. Создаём пустой список C для результата
3. Проходим по всем индексам:

   * берём A[i] и B[i]
   * складываем и записываем в C
4. Возвращаем C

### Псевдокод:

```
Вход: A, B
Выход: C

C = пустой список

для i от 0 до длина(A)-1:
    S = A[i] + B[i]
    добавить S в C

вернуть C
```
